# 망고아이 웹(mangoiweb)에 AI 상담직원 연동 가이드

대상 저장소: https://github.com/navy111p-sudo/mangoiweb
구조: Node/Express(`server.js`) + `public/` 정적 페이지. 메인 페이지는 `public/index.html`.

연동 방식: 아바타는 **별도 Cloudflare Pages**로 띄우고, 망고아이 웹의 HTML 페이지에
**iframe 위젯**(우측 하단 토글)을 한 줄 붙여 연결합니다. 모바일 웹에서도 그대로 동작합니다.

---

## 1단계 — 아바타 먼저 Cloudflare에 배포
이 폴더(`mangoi-develop2`)의 `README.md` 3~4번을 따라 배포하면
`https://OOO.pages.dev` 주소가 나옵니다. (GEMINI_API_KEY는 Cloudflare 환경변수에만 입력)

## 2단계 — mangoiweb 에 위젯 삽입
방법 A (간단): `public/index.html` 의 `</body>` 바로 위에 같은 폴더의
`embed-snippet.html` 내용을 붙여넣고, iframe `src` 를 1단계 주소로 바꿉니다.

방법 B (그대로 교체): 이 폴더의 `integration/mangoiweb-public-index.html` 가
위젯이 이미 삽입된 완성본입니다. iframe `src` 주소만 바꿔서 mangoiweb 의
`public/index.html` 로 덮어쓰면 됩니다.

> 화상통화(`/video-call`) 등 다른 페이지에도 상담직원을 띄우려면, 그 페이지의
> HTML `</body>` 위에도 같은 위젯 코드를 붙이면 됩니다.

## 3단계 — mangoiweb 커밋 & 푸시 (PC 터미널에서)
```bash
cd <mangoiweb 폴더>
git add public/index.html
git commit -m "feat: 망고아이 AI 상담직원 위젯 연동"
git push origin main
```
mangoiweb 의 기존 배포 파이프라인(`.github/workflows`, `deploy.bat`/`deploy.ps1`)이
자동으로 반영합니다.

---

## 왜 이렇게 하나 (요약)
- mangoiweb 은 Socket.IO 기반 Node 서버라 Cloudflare Pages 단독으로는 못 돌아갑니다.
  그래서 아바타만 가벼운 Pages 앱으로 분리하고, 웹에는 iframe 으로 얹는 게 가장 안전합니다.
- 키(GEMINI_API_KEY)는 아바타 쪽 Cloudflare 환경변수에만 두므로 망고아이 웹 코드에는
  키가 전혀 들어가지 않습니다.
- 위젯 한 조각만 붙이므로 mangoiweb 의 기존 화상통화/시그널링 기능에 영향이 없습니다.
